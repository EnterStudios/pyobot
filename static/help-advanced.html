If you feel basic operation of Pyobot is easy for you, you might be ready to dive deeper into what the bot can provide. Don't worry, we've got you covered.

Custom notifications format

A word of warning: this is more advanced topic that is completely optional. If you feel that the default one is enough for you, feel free to ignore this chapter of the Help.

Enabling/disabling custom notification format

To enable the feature, message the bot with following command: !enableCustomFormat 1.

Similarly, to disable it send: !enableCustomFormat 0.

Setting custom format

For Discord, there are two field available: title and body, while Facebook Messenger has only one available: body. Therefore, the command to set the title is Discord-exclusive and Facebook Messenger users have to put everything they want to see in the body field.

Syntax

Custom title: !setCustomTitle <custom title format>.

Custom body: !setCustomBody <custom body format>.

Available variables

  • %attack% - if IVs are available for a given spawn, this will return Attack value of the Pokemon. Sample value: 5;
  • %defence% - if IVs are available for a given spawn, this will return Defence value of the Pokemon. Sample value: 3;
  • %stamina% - if IVs are available for a given spawn, this will return Stamina (HP) value of the Pokemon. Sample value: 15;
  • %move1% - if moves are available for a given spawn, this will return first Move of the Pokemon. Sample value: Lick;
  • %move2% - if moves are available for a given spawn, this will return second Move of the Pokemon. Sample value: Blizzard;
  • %gender% - this will return gender symbol of the Pokemon or empty if genderless. Sample value: â™€;
  • %id% - this will return ID of the Pokemon. Sample value: 151;
  • %cp% - if available for a given spawn, this will return CP of the Pokemon. Sample value: 1230;
  • %iv% - if IVs are available for a given spawn, this will return sum of IVs the Pokemon. Sample value: 45;
  • %ivPercent% - if IVs are available for a given spawn, this will return percentage value of IV of the Pokemon. Sample value: 58%;
  • %suburb% - if available, will return suburb location data of the Pokemon. Sample value: Becontree Heath;
  • %postcode% - if available, will return suburb location data of the Pokemon. Sample value: RM9;
  • %niceName% - this will return the full name of the Pokemon (Unowns will have their letter form added). Sample value: Snorlax;
  • %remainingTime% - this will return time left to catch the Pokemon, in minutes and seconds format. Be aware that it will only be valid at the time the notification was sent. Also known as Time To Despawn. Sample value: 23:15;
  • %expiryTime% - this will return time when the Pokemon will despawn, in hour, minutes and seconds format. Also known as Time Of Despawn. Sample value: 10:12:23;
  • %location% - this will return postcode + suburb if available, otherwise just postcode. Sample value: RM9 Becontree Heath;
  • %locationMapUrl% - link to Google Maps with the location of the Pokemon. Sample value: https://www.google.com/maps/place/51.48376259,-0.24130472;
  • %closestStationName% - if available, this will return closest Tube, DLR or railway station of the Pokemon. The maximum distance between the station and the Pokemon is 500 metres in a straight line. Sample value: West Ealing;
  • %distance% - if you set your location, this will return the distance between your location and the Pokemon in kilometres. Sample value: 0.35;

Sample

Executing the following command on Discord:

  • !enableCustomFormat 1
  • !setCustomTitle %niceName% %ivPercent% %gender%
  • !setCustomBody Expires: %expiryTime% | %distance%km | %closestStation% | %postcode%

I found a bug, what to do now?

First of all - sorry for that. We are striving to provide you with the best experience possible, but unfortunately, as always with the software, errors do happen.

If you believe you found a bug - a command didn't work for you or a notification was missing for example - come to Discord and post about your problem on the #dev channel.
